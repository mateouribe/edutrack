      4. Key Functionalities
      1. User Management: The system categorizes users into different roles, such as students, teachers, and admins. Each user type has unique privileges. Students can attend classes, participate in events, and view user profiles. Admins and teachers have elevated permissions, enabling them to create, edit, and delete classes, events, and user profiles.
      2. Class Management: EduTrack allows the creation and management of class records, each with details such as class name, description, room number, and scheduling information. This functionality is crucial for organizing the academic structure.
      3. Event Management: The platform includes an event management module. Events can be academic or extracurricular, with attributes like title, description, date, cost, type, location, and duration. This feature enriches the educational experience by facilitating diverse activities.
      4. Relational Data Handling: The database architecture incorporates tables for users, classes, events, and their interrelations, ensuring seamless management of the many-to-many relationships between users and classes/events.
      5. Admin User Creation: A special feature for creating an admin user is included, setting a precedent for system security and administrative control.
      6. Database Flexibility: EduTrack includes functionality to drop the database, allowing for system resets or restructuring as needed.
